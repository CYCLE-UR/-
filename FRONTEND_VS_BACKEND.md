# 前台 vs 後台 功能對比

## 核心差異

### 🎫 前台系統 (demo.html)
**定位：** 普通用戶使用的搶票工具

**特點：**
- ✅ 匿名登入，無需密碼
- ✅ 只能看到自己的任務
- ✅ 創建、管理個人搶票任務
- ✅ 查看個人統計數據
- ✅ 接收個人通知

**功能範圍：**
- 個人任務管理（CRUD）
- 個人統計分析
- 通知設置
- 本地數據存儲

---

### 🔐 後台系統 (admin.html)
**定位：** 管理員使用的系統管理工具

**特點：**
- 🔒 需要管理員密碼登入（默認：admin/admin123）
- 🌐 可查看所有用戶的數據
- 📊 全局統計和監控
- 🛠️ 系統管理功能
- ⚠️ 危險操作權限

**獨有功能：**

#### 1. 管理員認證
- 登入界面與密碼保護
- 會話管理
- 安全退出

#### 2. 全局數據視圖
- 查看所有用戶列表
- 查看所有任務（跨用戶）
- 用戶活動分析
- 用戶分類（重度/中度/輕度/新用戶）

#### 3. 系統監控（獨有）
- 存儲使用情況（MB、項目數）
- 系統性能指標（平均執行時間）
- 系統健康狀態
- 用戶活動分布圖

#### 4. 高級統計
- 跨用戶的成功率統計
- 熱門活動排行（全局）
- 時段分布分析（全局）
- 7 日趨勢圖（全局）

#### 5. 批量管理
- 批量刪除任務
- 批量暫停/恢復
- 用戶數據導出（CSV）
- 全局數據導出（JSON）

#### 6. 危險操作
- 清理舊任務（30 天前）
- 重置整個系統
- 強制刪除任何用戶的任務

#### 7. 統計卡片擴展
- 前台：4 個基礎統計
- 後台：8 個進階統計（包含失敗數、待執行數、存儲用量等）

---

## 視覺差異

### 前台界面
- 溫暖色調（灰白為主）
- 用戶友好的卡片式設計
- 強調個人操作體驗
- 簡潔的導航

### 後台界面
- 專業色調（帶紅色警告標識）
- 管理員登入頁面（漸變背景）
- 側邊欄導航（7 個模塊）
- 頂部顯示「管理員模式」提示
- 危險操作紅色標識

---

## 數據權限對比

| 功能 | 前台 | 後台 |
|------|------|------|
| 登入方式 | 匿名（自動） | 密碼驗證 |
| 數據範圍 | 僅個人 | 全局（所有用戶） |
| 任務查看 | 只看自己 | 看所有人 |
| 用戶管理 | 無 | 完整列表 + 詳情 |
| 系統監控 | 無 | 完整監控面板 |
| 批量操作 | 無 | 支持 |
| 危險操作 | 無 | 完整權限 |
| 數據導出 | 個人數據 | 全局數據 |

---

## 使用場景

### 前台適合：
- 🎤 追星族：搶演唱會門票
- 🏃 普通用戶：個人任務管理
- 📱 移動端使用：簡單操作

### 後台適合：
- 👨‍💼 系統管理員：監控整體運行
- 📊 數據分析師：查看全局統計
- 🔧 運維人員：系統維護和優化
- 🛡️ 安全人員：用戶行為審計

---

## 安全機制

### 前台
- LocalStorage 存儲個人數據
- 只能訪問自己的任務
- 無敏感操作權限

### 後台
- 需要管理員密碼
- 會話管理（localStorage 標記）
- 全局數據訪問權限
- 危險操作二次確認
- 紅色警告提示

---

## 技術實現差異

### 前台 (demo.html)
```javascript
// 只獲取當前用戶的任務
const tasks = await botEngine.getAllTasks()  // 自動過濾當前用戶
```

### 後台 (admin.html)
```javascript
// 獲取所有用戶的任務
const tasks = await adminSystem.getAllUsersTasksAdmin()  // 遍歷 localStorage
```

---

## 總結

前台和後台現在有明確的職責分工：

**前台 = 用戶工具** → 專注個人搶票體驗  
**後台 = 管理平台** → 專注系統監控和管理

這符合標準的 Web 應用架構模式。
